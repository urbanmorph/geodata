geodata

Open-source visualiser for India's geo data.

Find, view, slice and contribute India's official admin-boundary datasets — and community-contributed geo layers — without an account, without an API key, without a single SQL query.

What makes this different

India has plenty of geo data. The hard part has always been finding it, validating it, and adding to it.

The official portals lock data behind logins, restrict derivatives, or require Windows plugins. The community efforts — DataMeet, the dozens of GitHub scrapes — sit in repos with no preview, no validation, no way for anyone else to add. The drag-and-drop viewers (geojson.io, Kepler, the late Placemark) handle a single file at a time and choke past about 8,000 features. None of them combine catalog, viewer, validator, and contribution flow under one roof — and none of them do it for India specifically.

geodata is that combination:

  • A visual catalog — every state, district, subdistrict, block and village under LGD codes, with side-by-side comparison of SOI, Bhuvan and geoBoundaries. Each layer renders on a map and downloads as Parquet, GeoJSON or KML.
  • A drag-drop verifier — drop any file (GeoJSON, KML, KMZ, Parquet, up to 500 MB) and the browser tells you what's wrong before it leaves your machine. Wrong CRS, broken geometries, features outside India — flagged inline.
  • Anonymous contribution — submit your own layer with no signup, no email, no API key. The site returns an admin token (saved to your browser, downloaded as a backup .txt) that you keep forever. Auto-moderation handles licence, attribution and validity server-side. Open licences only.
  • Mixed catalog — accepted submissions appear in the main catalog within a build cycle, badged [community], sortable by recency or thumbs-up rating. Curated and community layers share the same view and download surface.

Everything is open: the code under MIT, curated data under each upstream's open licence (CC0-1.0, CC-BY-4.0, GODL-India), community submissions under whichever open licence the contributor picks. No login, no tracking cookies, no third-party analytics.

Who is it for

  • Journalists — pull a state's district boundaries in ten seconds for a story map.
  • Researchers — slice 584,615 village polygons by state, export as GeoJSON or Parquet.
  • Civic technologists — share city-scale infrastructure datasets (bike lanes, parks, polling booths) with attribution and download.
  • Educators — render any of India's 7,146 community-development blocks on a map for a classroom demo.
  • App developers — direct PMTiles URLs, no rate limits, no API key.

What you can do today

  • View every admin level from state down to village on a map.
  • Filter any LGD-coded layer by state and instant-download as Parquet, GeoJSON, or KML (Google Earth ready).
  • Cross-compare LGD, SOI, Bhuvan and geoBoundaries side by side.
  • Preview a file — drag-drop a GeoJSON / KML / KMZ / Parquet (up to 500 MB) and see it on a map, validate it, and optionally publish under an open licence in seconds. No signup, no email; you get an admin token you can keep forever.

How it works

Static site on Cloudflare Pages · data on R2 · in-browser DuckDB-WASM for slicing · MapLibre + PMTiles for rendering · Pages Functions + D1 SQLite for the contribution flow. Code on GitHub. No tracking, no third-party analytics, no fingerprinting.

Where the curated data comes from

LGD Local Government Directory — authoritative, with full code chain. CC0-1.0 / CC-BY-4.0 SOI Survey of India open-data series, community-republished. CC0-1.0 Bhuvan NRSC/ISRO Bhuvan panchayat portal. CC0-1.0 PMGSY Pradhan Mantri Gram Sadak Yojana — rural roads, blocks. CC0-1.0 geoBoundaries Independent cross-check, name-only attributes. CC-BY-4.0

Use of data

  • Provided as-is, no warranty. Express or implied guarantees of accuracy, completeness or fitness for a particular purpose are not made by this site or its maintainers.
  • Boundaries are for reference, not legal authority. For official, administrative or land-record use, refer directly to LGD, SOI, Bhuvan or the state revenue department.
  • Curated layers reflect the upstream source at fetch time — see the freshness pill on each card. Upstream sources change; the catalog can lag.
  • Community submissions are auto-moderated, not editorially curated. The platform checks licence, format and basic geometry validity. It does not verify accuracy, completeness or authorship beyond the contributor's self-attestation. Verify provenance via the source link on each card before relying on community data.
